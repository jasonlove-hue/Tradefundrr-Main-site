## ⚠ Guarantee Removal Instructions ⚠

The "Hit your first payout. Get your eval fee back." guarantee block was shipped at Jason's direction with the explicit note that **it may need to be removed** depending on business viability. The implementation is wrapped in clean START/END markers throughout so removal is a clean three-step operation.

### To remove the entire guarantee:

**Step 1 — Remove the primary banner from `index.html`**
- Open `index.html`
- Find the comment marker `<!-- GUARANTEE_BANNER_START -->` (inside `<section class="plans">`, between the section header and the pricing grid)
- Delete everything from that marker through `<!-- GUARANTEE_BANNER_END -->` (inclusive)

**Step 2 — Remove the per-card badges from `index.html`**
- Inside each of the three pricing cards (Options, Stocks, Futures), find this pattern:
  ```html
  <!-- GUARANTEE_BADGE_START --><span class="stack-card__guarantee">Refunded with first payout</span><!-- GUARANTEE_BADGE_END -->
  ```
- Delete each occurrence (three total, one per card)
- A quick grep: `grep -n "GUARANTEE_BADGE" index.html` will surface all three lines

**Step 3 — Remove the CSS block from `mobile-fixes.css`**
- Open `assets/css/mobile-fixes.css`
- Find `/* === GUARANTEE START === */`
- Delete everything from that marker through `/* === GUARANTEE END === */` (inclusive)

**Step 4 — Bump the cache-buster on `mobile-fixes.css`**
- In `index.html`, find the line:
  ```html
  <link rel="stylesheet" href="assets/css/mobile-fixes.css?v=YYYYMMDDHHMM" ...>
  ```
- Update the `?v=` query string to a fresh timestamp so browsers re-fetch the updated CSS

**Step 5 — Commit and push**
```powershell
git add index.html assets/css/mobile-fixes.css
git commit -m "Remove payout guarantee block + per-card badges"
git push origin main
```

Total removal time: under 5 minutes if done carefully. No other section depends on the guarantee, so nothing else breaks.
