## ⚠ Workspace Notes — Critical for Next Session

### Edit-tool size cap (~91 KB) — STILL IN EFFECT

`stocks-preview-v2.html` is now **175 KB** and `pages/futures/index.html` is **116 KB**. Both are well over the ~91 KB cap where the harness's `Edit` tool silently truncates. **All edits to these files this session went through `python3` heredoc via `mcp__workspace__bash`, never the Edit tool.** Keep doing that. The Edit tool is fine only for small (<50 KB) files.

### pages/futures/index.html was truncated (now fixed)

On opening it to add animation, the file ended mid-tag: `})();\n  <` with no closing `</script></body></html>` and one `<script>` left unclosed (3 opens / 2 closes). This is the classic Edit-tool size-cap truncation the 05-28 handoff warned about, from an earlier session. Repaired by closing the dangling script, appending the animation script, and adding `</body></html>`. **Verify file integrity (closing tags, script balance) on any large HTML you touch — re-run the grep checks before AND after.**

### File-integrity verification protocol (used all session)

After every bash write to a large file:
1. `grep -c "</body>"` and `grep -c "</html>"` both return 1.
2. `<script>` opens == `</script>` closes.
3. `<section>` opens == `</section>` closes; spot-check `<div>` / `<details>` balance with a quick python count.
4. `tail -3` shows the natural end of file.
