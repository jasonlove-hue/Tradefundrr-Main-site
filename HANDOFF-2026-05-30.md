## ⚠ Workspace Notes — Critical for Next Session

### OneDrive + git lock conflicts
The repo lives in a OneDrive-synced folder, so OneDrive keeps re-locking files inside `.git` (`index.lock`, `HEAD.lock`) as git creates them, breaking `git add` / `commit`. Workaround used all session: **pause OneDrive syncing**, then `del /s /q ".git\*.lock"`, then commit. **Long-term fix: move this repo out of OneDrive** (e.g. `C:\dev\tradefundrr-main-site`).

### Bash mount serves STALE / TRUNCATED copies
The Linux bash mount showed `stocks-v2-redesign.html` as **356 of its real 1621 lines**, and showed null-byte padding on freshly-written files. **Do NOT read or write the big HTML files through bash/cat/python/sed** — use the authoritative `Read` / `Write` / `Edit` file tools, which see the true files. `git`/`grep`/`ls` on the mount are usually fine; reading/writing OneDrive-synced HTML content is not.

### Overlaying HTML on a transparent vault PNG (cascade technique)
- Use **natural image sizing** on the vault `<img>` (`width:100%; height:auto; position:relative`). Do **NOT** use `object-fit:contain` — it letterboxes the image inside the box, so percentage-positioned overlays (the cavity) drift off the opening. This caused several wrong-position rounds.
- Calibrate the opening center with a **labeled coordinate grid** rendered via ImageMagick, then **verify with a static IM composite** (dark cavity + a few cards) before trusting the numbers. Downscaled preview reads are unreliable — they cost ~6 wrong guesses today.
- The AI vault came out on a transparent background but with a **faint glow padding** that `-trim` won't cut and that makes the vault look small; crop to the solid bbox (`-fuzz -trim` gives it) **with margin so you don't clip the door.** The first crop lopped the door's right edge — that was the "door cut off."
- A `.media` override needs **higher specificity** (`.media.media`) than the page's later `left:65%` rule, or it silently loses.

### Live-tuning trick
You can iterate position/size **without a deploy cycle**: open the live page in the Chrome MCP, inject a `<style>` with the candidate values (high specificity), `getBoundingClientRect` the opening vs the text, screenshot, then apply the confirmed numbers to the file. Used to nail the door-clearance values.
